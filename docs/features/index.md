# Índice de features

Especificaciones de funcionalidades (spec-driven). Cada feature debe tener:

- **Descripción** — Qué hace y por qué.
- **Requisitos** — Funcionales, técnicos, dependencias y restricciones.
- **Criterios de Aceptación** — Qué debe cumplirse para darla por hecha.
- **Implementación** — Enfoque técnico, arquitectura y detalles clave.

|--------|-----|--------|
| *(ninguna documentada aún)* | — | — |

## Crear una nueva spec

1. Crear `docs/features/NombreFeature.md` (PascalCase).
2. Rellenar las 4 secciones anteriores.
3. Añadir la fila correspondiente en esta tabla.
